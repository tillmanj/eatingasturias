*Jekyll files for building the Eating Asturias public site*:

[Eating Asturias](https://eatingasturias.com) is a website about food and food culture in the Asturias region of Spain. It is generated from [Markdown](https://daringfireball.net/projects/markdown/) files by the [Jekyll](https://jekyllrb.com/) static site generator (SSG). 

I encourage everyone to participate in improving the content on the site. I am open for corrections and new content submissions. This website is [developed in the open](https://github.com/tillmanj/eatingasturias/wiki/Developed-in-the-Open).

## Corrections
Please use the *Issues* link above to suggest a correction. 

## New Content
For new content you would like to see on the site, you have two options:
- Use the *Issues* link above to suggest an article for the site that you would like to see me write
- Create a pull request with an article you have written yourself

## How To Create A Pull Request
To make changes to the site, a *pull request* is required. See [About pull requests](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests#draft-pull-requests)

Please follow the instructions on Github Docs for [creating a pull request from a fork](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request-from-a-fork).

#### Install The Site Locally (Fork or Clone)
*See [Building The Site Locally](https://github.com/tillmanj/eatingasturias/wiki/Building-The-Site-Locally) for complete instructions*

#### Write Your Article
Using the Markdown tool or IDE of your choice, write the article you want added to the site.

#### Submit Your Pull Request
To create a pull request that is ready for review, click Create Pull Request. To create a draft pull request, use the drop-down and select Create Draft Pull Request, then click Draft Pull Request.

#### Wait Patiently
After you create your pull request, I will review the request, and respond with questions, suggestions, reasons why I am rejecting it, or a congratulations message if I accept the pull request without modification. It will be published on the site and you will be credited as the author.
